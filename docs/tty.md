## Транспорт

Интерфейс: виртуальный COM-порт (USB CDC).

### Настройки порта (рекомендуемые):

* скорость: 115200 бод

* биты данных: 8

* чётность: нет (N)

* стоп-биты: 1

Кодировка: ASCII, одна строка — одно событие.

## Формат сообщения

Обычно одно событие — одна строка вида:
```
EVT,<input_id>,<state>\n
```
Поля:

* `EVT` — тип сообщения (от англ. event).
Зарезервировано под события с цифровых входов.

* `<input_id>` — номер входа (десятичное целое число 1-4).

* `<state>` — состояние входа:

* `1` — ошибка / тревога / активное состояние.

* `0` — норма / ошибка исчезла / вход вернулся в неактивное состояние.

* `\n` — конец строки (символ перевода строки, LF, код 0x0A).
При желании можно использовать \r\n, если так проще на МК.

## Примеры
```
EVT,3,1\n
```
На входе 3 появилась ошибка (или уровень перешёл в активное состояние).
```
EVT,1,1\n
```
Активировалась ошибка/тревога на входе 1.
